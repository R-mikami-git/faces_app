本アプリの使い方
１、ターミナルに[python app.py]で利用。
２、モデルの初期化はターミナルに[python models_init.py]で実行。
３、ログイン画面はユーザーとパスワードで入れる。
    ホーム画面→登録→クラス[000000]と入力すると、ログイン画面に入れるアカウントになる。
    パスワードは登録したクラス名[000000]を使用すること。

本アプリの課題
１、あまりモデルの精度がよろしくない。
    少し顔を隠すだけでエラーになるし、顔を遠ざけるだけでも失敗する。
２、モデルがオンライン学習に対応していない。
    本来は毎日の出退席のデータも学習し精度向上に繋げるべきだがそのようなシステムはない。
３、セキュリティが甘い。
    パスワードがそもそも極めて脆弱。ハッシュ値も何も使わず平文の[000000]。ありえない。
    また、ログイン画面からバックした後、再びログインしなければ設定画面に入れないなどの工夫も必要だろう。

本アプリ作成の反省
１、技術不足の為、設計を簡素に行い欲しい機能を随時追加する開発方式を採用。
    どのような機能をどのように作っていけばよいかその都度学べた為、学習用としては良かった。
    しかし、想定通りではあるが、設計をおろそかに後から機能を追加する方式は極めて非効率で、あらゆる部分が中途半端になった。
    本アプリを本番稼働想定で作成するのなら、まず高い精度のモデルの作成と必要なセキュリティシステムの設定から始めるのが良いだろう。
